# Spring Cloud学习笔记-7.服务网关（2）

定义Pre Filter

  • 例子：定义一个检查token请求参数的pre filter
@Component
public class TokenFilter extends ZuulFilter {


    @Override
    public String filterType() {
        return PRE_TYPE;
    }

    @Override
    public int filterOrder() {
        return PRE_DECORATION_FILTER_ORDER - 1;
    }

    @Override
    public boolean shouldFilter() {
        return true;
    }

    @Override
    public Object run() {

        RequestContext requestContext = RequestContext.getCurrentContext();
        HttpServletRequest request = requestContext.getRequest();

        String token = request.getParameter("token");
        if (StringUtils.isEmpty(token)) {
            requestContext.setSendZuulResponse(false);
            requestContext.setResponseStatusCode(HttpStatus.UNAUTHORIZED.value());
        }
        return null;
    }
}

定义Post Filter

  • 定义一个post filter，在response的header里添加参数
@Component
public class AddResponseHeaderFilter extends ZuulFilter {
    @Override
    public String filterType() {
        return POST_TYPE;
    }

    @Override
    public int filterOrder() {
        return SEND_RESPONSE_FILTER_ORDER - 1;
    }

    @Override
    public boolean shouldFilter() {
        return true;
    }

    @Override
    public Object run() {
        RequestContext requestContext = RequestContext.getCurrentContext();
        HttpServletResponse response = requestContext.getResponse();

        response.setHeader("X-Foo", UUID.randomUUID().toString());

        return null;
    }
}

限流

  • 定义Pre Filter，优先级通常设为最高，在认证鉴权之前
@Component
public class RateLimitFilter extends ZuulFilter {

    // Google的令牌桶算法
    private static final RateLimiter RATE_LIMITER = RateLimiter.create(100);

    @Override
    public String filterType() {
        return PRE_TYPE;
    }

    @Override
    public int filterOrder() {
        return SERVLET_DETECTION_FILTER_ORDER - 1;
    }

    @Override
    public boolean shouldFilter() {
        return true;
    }

    @Override
    public Object run() {
        if (!RATE_LIMITER.tryAcquire()) {
            throw new RateLimitException();
        }
        return null;
    }
}

Spring Cloud学习笔记-7.服务网关（1）

服务网关重要关注点

  • 高可用性
  • 并发性能
  • 安全性
  • 扩展性

常见网关方案

  • Nginx + Lua：性能极高，高并发，扩展性好
  • Kong：基于Nginx + Lua，提高易用性，但是是商业收费方案
  • Tyk：支持配额和速度限制，支持认证、数据分析、多租户、多组织，支持全Restful API。用go开发，性能和扩展性不错
  • Spring Cloud Zuul：以Java技术栈为主的话，提供了认证、鉴权、限流、动态路由、监控、弹性、安全、负载均衡、协作单点压测、静态响应等边缘服务的框架。一代的Zuul在性能上不具备优势，第二代据说有很大提升，与Spring Cloud配合好，路由+过滤器为主要功能，核心是一系统的过滤器
  • 各种方案可以根据项目实际情况分层分段混搭

Zuul的四种过滤器API

  • 前置（Pre）：常用于限流、鉴权、参数校验、请求转发
  • 路由（Route）
  • 后置（Post）：常用于统计、日志
  • 错误（Error）

启动Zuul

  • 引入工程依赖
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-netflix-zuul</artifactId>
        </dependency>
  • bootstrap.yml里配置环境
spring:
  application:
    name: api-gateway
  cloud:
    config:
      discovery:
        enabled: true
        service-id: CONFIG
      profile: dev
eureka:
  client:
    service-url:
      defaultZone: http://localhost:8761/eureka/
  • 主类上启动@EnableZuulProxy注释

路由管理

  • 在bootstrap.yml文件里增加路由
zuul:
  routes:
    myProduct:
      path: /myProduct/**
      serviceId: product

简单写法

      product: /myProduct/**
  • 允许查看路由信息，在bootstrap.yml文件里增加如下配置，然后访问：http://ip:port/application/routes
management:
  security:
    enabled: false
  • 排除某些路由
  ignored-patterns:
    - /url1
    - /**/product/list
  • 传递Cookie，将sensitiveHeaders设为空即可
zuul:
  # 增加路由
  routes:
    myProduct:
      path: /myProduct/**
      serviceId: product
      # 传递cookie
      sensitiveHeaders:
  • 动态路由，请建一个类，刷新路由配置
@Component
public class ZuulConfig {

    @ConfigurationProperties("zuul")
    @RefreshScope
    public ZuulProperties zuulProperties() {
        return new ZuulProperties();
    }
}

zuul高可用

  • 多个zuul节点注册到Eureka Server
  • Nginx和zuul混搭

Spring Cloud学习笔记-6.消息和异步处理

使用RibbitMQ的三种方法

  • pom里引入
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-bus-amqp</artifactId>
</dependency>

  1. Java类中加入注解@RabbitListener(queues = "myQueue")

  2. 如需RabbitMQ自动创建对应的queue，则需要使用@RabbitListener(queuesToDeclare = @Queue("myQueue"))

  3. 自动创建，Exchange和Queue绑定。这种情况通常用在有多个不能的消息topic时，不同的进程消费不同的topic

@RabbitListener(bindings = @QueueBinding(
        value = @Queue("myQueue"),
        key = "myKey",
        exchange = @Exchange("myExchange")
))

使用Spring Cloud Stream

  • 支持的MQ有两种：RabbitMQ和Kafka

  • pom.xml里引入

        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-bus-amqp</artifactId>
        </dependency>

        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-stream-rabbit</artifactId>
        </dependency>
  • 设置消息队列名及消息格式，便于定位问题时，直接在rabbitMQ控制台可以直接看到消息内容，在bootstrap.yml里增加配置项如下：
  cloud:
    stream:
      bindings:
        myMessage:
          group: order
          content-type: application/json
  • 如何要设置回复消息队列，可以使用@SendTO注解

Spring Cloud学习笔记-5.config配置中心

使用方法

1.pom.xml引入依赖

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-config-server</artifactId>
</dependency>

2.项目主启动文件增加@EnableDiscoveryClient和@EnableConfigServer注解，完成服务注册和开启config server服务

3.在application.yml指定git配置路径

spring:
  application:
    name: config
  cloud:
    config:
      server:
        git:
          uri: https://gitlab-demo.com/6003575/my-config-repo.git
          username: 6003575
          password: abcd1234
          basedir: /Users/daiqiang/Dev/java/springcloud_sell/config-repo
eureka:
  client:
    service-url:
      defaultZone: http://localhost:8761/eureka/
  • 可以通过http://localhost:8082(服务指定端口号)/filename-a.yml文件来检验是否从配置服务器获取到指定配置
  • 文件名有规则，如：/{label}/{name}-{profiles}.yml，label指分支，name为原文件名，profiles为环境，后面为扩展名
  • 本地git仓库路径可以通过basedir参数指定
  • 服务名+环境名的组合中，服务名.yml里存放多种环境公共的配置，服务名+环境名.yml文件中放差异性的配置，config client加载时，会合并主配置文件与当前环境配置文件的内容

4.其它服务要使用配置中心，需要做如下修改：

  • application.yml改名为bootstrap.yml
  • 在bootstrap.yml文件中加入配置，然后将需要放入配置中心的配置项移动到git配置文件中
spring:
  cloud:
    config:
      discovery:
        enabled: true
        service-id: CONFIG
      profile: dev

Spring Cloud学习笔记-4.Ribbon应用间通信

HTTP VS RPC

  • Dubbo定位是RPC框架，可以作为Spring Cloud二进制通信方案
  • Spring Cloud是微服务架构下的一站式解决方案

客户端负载均衡器：Ribbon

  • RestTemplate
    • 使用Rest方式调用其它服务
  • Feign
    • 声明式REST客户端
    • 采用接口+注解的方式，即定义一个Interface，然后在上面加上注解
    • 使用时感觉不到是在调用REST，像是在用本地接口，底层还是使用了Ribbon的机制
  • Zuul

Ribbon主要能力

  • 服务发现，根据服务名字，找出所有同名实例
  • 服务选择规则，依据规则策略，从多个服务中选择有效的服务
  • 服务监听，检测无效的服务，高效去除

REST Template用法

  1. 直接使用RestTemplate，url写死
  2. 用LoadBalancerClient通过应用名称获取url，然后再使用RestTemplate
  3. 用@LoadBalanced，可在restTemplate里使用应用名字

Ribbon主要组件

  • ServerList，获取所有的可用服务
  • ServerListFilter：过滤到一部分地址

  • IRule：选择一个实例

    • 在application.yml里配置选择策略，如：选择随机策略
    ORDER:
    ribbon:
        NFLoadBalancerRuleClassName: com.netflix.loadbalancer.RandomRule

Feign使用方法

1.增加包依赖

        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-feign</artifactId>
        </dependency>

2.启动类增加@EnableFeignClients注解
3.新建Client Interface，增加@FeignClient(name = "服务名")注解
4.在方法前增加@GetMapping("/方法名")注解，明确被调用的方法，这个注解跟被调用端的ServerController的方法有对应关系，要注意是完整路径对应，不要遗漏被调用类里@RequestMapping定义的路径
